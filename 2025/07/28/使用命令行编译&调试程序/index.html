使用命令行编译&调试程序

编译程序

g++用法与参数介绍

我使用g++来编译c++程序。

基本格式：g++ text.cpp 。默认生成a.exe可执行文件。

在此基础上可以添加其他参数来扩展功能。各个参数之间可以自由搭配组合，对先后顺序没有要求。

  1. g++ text.cpp -o text ：编译text.cpp文件并且生成text.exe可执行文件，-o就是在引出可执行文件名称。但要注意千万不要在-o后面写text.cpp！

  2. g++ text.cpp -std=c++{11,14,17} ：编译时指定c++标准，在OI考试中默认c++14，所以了解即可。

  3. g++ text.cpp -Wall：给代码提供基础警告，包含未使用的变量、未初始化的变量、隐式类型转换、函数无返回值、指针类型不匹配等等。需要注意名字里的all是历史遗留，实际上并不包括所有警告。

  4. g++ text.cpp -Wextra：给代码提供额外警告，可排查if（a=b）这样的奇怪语句，有符号与无符号之间的比较等等。常与-Wall搭配使用，这样就可以排查所有警告了。

  5. g++ text.cpp -Werror：将警告视为报错。在不与-Wall或-Wextra联用的情况下没有意义。

  6. g++ text.cpp -g：这样生成的可执行文件才可以进行GDB调试。

总结：在OI中，直接使用g++ text.cpp -o text -Wall -Wextra -g就可以啦！

GDB调试

使用GDB调试效率十分之高，是一个值得掌握的技能。

首先我们拿到一个可执行文件（编译时带-g参数），然后在命令行输入gdb text即可进入调试界面。下面来介绍GDB的命令。

  1. file：修改文件后不想退出GDB？直接在gdb内make，然后file一下可执行文件就可以了。可以选择保留或不保留停止点之类的东西，但要注意行号的变化。

  2. run(r)：从头开始运行程序，直到遇到停止点。

  3. break(b)：在行数或函数的开头设置一个断点（不执行那一行或那个函数）。此外，还可以添加第二个参数——bool表达式。例如：b 8 if a==8 && b==7 。十分神奇。

  4. clear(cl)：清除断点，加了行数或函数名就是清空那里的断点。

  5. delete(d)：删除停止点，后加ID。

  6. continue(c)：在程序中断后继续执行程序，直到遇到下一个停止点。

  7. next(n)：执行一条语句，如果有函数调用就直接执行完这个函数。

  8. step(s)：执行一步，如果有函数就跳到函数内部执行。

  9. skip：在s时不进入某个函数内部，但会执行这个函数，效果和n类似。删除时使用skip delete <skip ID>：删除这个skip规则。

  10. jump：跳跃至某一行。但要注意不同函数间的jump可能导致函数栈逻辑错误。

  11. print(p)：打印变量或表达式的值。

  12. display(disp)：在每一次程序停下时，都打印display的变量或表达式的值。删除时使用delete display <ID>。

  13. finish：在函数（除main函数）内部时使用，运行直到该函数停止。

  14. watch(wa)：在变量被写时停止，并输出变化前后的值。

  15. awatch(aw)：在变量被读或写时停止。

  16. rwatch(rw)：在变量被读时停止。

  17. info(i)：查看信息。如info berak,info skip等等。注意 i b是查看所有停止点而非断点。
